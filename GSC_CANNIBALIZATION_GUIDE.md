## 2. カニバリ判定ロジック

### 2.1 重複クエリの判定基準

同一クエリが**複数ページで Position < 30** ならカニバリ候補。

| 状況 | 判定 | 対応 |
|------|------|------|
| クエリ A → ページ X (Pos 8), ページ Y (Pos 22) | カニバリ | X 本命・Y → X への内部リンク強化 |
| クエリ A → ページ X (Pos 12), ページ Y (Pos 35) | 正常 | Y は実質ランキング外 |
| クエリ A → ページ X (Pos 6), ページ Y (Pos 7) | **重大カニバリ** | コンテンツ統合 or canonical or 差別化 |

## 3. 検出後の対応パターン

### 3.1 補助ページ (Pos 22-30) → メインページへの内部リンク強化

**例**: 「福岡 ゴルフ 平日 安い」で
- book-fukuoka-cheap.html (Pos 12) ← 本命
- hub-budget.html (Pos 25) ← 補助

**対応**: hub-budget.html の冒頭〜中盤に book-fukuoka-cheap.html への目立つアンカーリンク (例: 「24 コース完全比較表はこちら →」)。アンカーテキストは KW を含める。

### 3.2 重大カニバリ (両方 Pos 6-15)

**判断分岐**:
| 状況 | 対応 |
|------|------|
| コンテンツ重複 70% 以上 | **コンテンツ統合** + 一方を 301 リダイレクト |
| 内容は違うが同じ KW で競合 | **canonical 修正** で本命を指定 |
| 役割が明確に違う (例: LP vs hub) | **コンテンツ差別化** で各ページの target 明確化 |

### 3.3 LP が全くランキングしていない (Pos 30+ ばかり / Impression が極小)

**判断**: コンテンツが弱い or 新規ゆえ評価まだ。

**対応選択肢**:
- 内部リンク強化 (現状: hub-budget から「Quick Booking Guides 3選」あり・他からも追加)
- E-E-A-T 強化 (取材コンテンツ追加・編集部評価コメント等)
- **待つ** (新規ページは順位安定まで 4-12 週間)

### 3.4 Phase 2 LP が予想外の KW で評価されている

**例**: book-fukuoka-tomorrow.html が「福岡 ゴルフ 平日 予約」で Pos 8 (想定 KW「当日予約」ではない)

**対応**: そのクエリでより最適化 (タイトル・H2 に該当 KW 自然挿入) → ただし観測期間終了後に実施
